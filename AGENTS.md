- **产品类型**: 金融策略可视化 Web 平台（工具+看板混合）

## 功能列表

### 页面：智能问答首页（HomePage / `/`）

- **页面目标**: 用户用自然语言提问，快速获得黄金对冲配置建议
- **功能点**:
  - **自然语言输入**: 大输入框 + 提交按钮，支持中英文输入，回车或点击提交
  - **语言自动识别**: 检测输入文本语言，用于后续 AI 解析和回复语言匹配
  - **参数解析**: 调用 AI 分类插件，从提问中提取行业、风险偏好、市场状态等结构化参数
  - **配置比例计算**: 基于解析出的参数，按内置策略公式计算建议黄金配置比例
  - **建议文案生成**: 调用 AI 文本生成插件，流式输出对冲逻辑说明和风险提示
  - **结果展示卡片**: 展示建议比例（大号数字突出）、对冲逻辑、风险提示三部分内容
  - **推荐提问示例**: 初始态展示 2-3 条推荐问题，点击可直接填入输入框

### 页面：行业配置计算器（CalculatorPage / `/calculator`）

- **页面目标**: 用户手动选择行业和市场参数，精确计算黄金配置比例
- **功能点**:
  - **行业选择**: 下拉选择器，13 个行业选项（能源、公用事业、必需消费、医药、金融、工业、材料、可选消费、科技、通信、地产、综合、大盘）
  - **风险偏好滑块**: 三档滑块（保守/稳健/激进），对应基准比例 15%/10%/5%
  - **市场状态调节**: 三组滑块/开关（波动：低/正常/高；通胀：低/高；恐慌：低/高），分别对应 +20%/+30%/+25% 调整
  - **实时计算**: 参数变化时立即重新计算，结果无延迟
  - **结果展示面板**: 显示建议黄金 ETF 配置比例（主数值）、预期对冲效果、下行保护强度
  - **计算公式透明化**: 展示计算过程（基准 × 行业系数 × 市场调整 = 结果），增强可信度

### 页面：策略回测看板（BacktestPage / `/backtest`）

- **页面目标**: 展示自适应策略的历史表现，对比静态对冲和纯股票
- **功能点**:
  - **KPI 指标卡片行**: 4 张卡片展示年化收益、最大回撤、夏普比率、下行保护率，对比三种策略
  - **收益对比曲线图**: ECharts 折线图，三条曲线（自适应策略 / 静态对冲 / 纯股票），线条平滑，支持 hover  tooltip
  - **市场状态分布饼图**: 两个环形饼图，分别展示高通胀/低通胀占比、高 VIX/低 VIX 占比
  - **图表交互**: 图例点击显隐曲线、hover 显示详细数据、自适应容器宽度

### 页面：研究结论（ResearchPage / `/research`）

- **页面目标**: 展示四大核心研究发现，用数据支撑策略逻辑
- **功能点**:
  - **四大发现卡片网格**: 2×2 布局，每张卡片含标题、数据支撑数值、简要说明
  - **行业异质性卡片**: 展示各行业对冲 Beta 差异，突出能源行业 0.367 最强
  - **状态依赖卡片**: 说明不同市场状态下对冲效果差异
  - **非对称对冲卡片**: 说明黄金在下跌市和上涨市的非对称表现
  - **避险边界卡片**: 说明黄金对冲的适用范围和局限性

### 全局功能

- **语言切换**: 顶部导航栏 EN/中文 切换按钮，切换后所有文案即时更新，偏好保存到 localStorage
- **响应式布局**: 桌面端多栏布局，移动端自动堆叠为单列
- **双语文案**: 所有界面文本均有中英两套，通过 i18n 机制管理

  /** 计算过程明细 */

    baseRatio: number;
    industryFactor: number;
    marketAdjustment: number;
  };
}

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free Direction —— 无参考材料，基于产品语义与用户需求自主建立视觉系统
- **核心情绪 / 应用类型**: 金融策略工具的专业可信与贵金属质感，数据驱动 + 决策辅助
- **独特记忆点**: 金色渐变描边 + 深蓝底的"贵金属仪表盘"质感，关键数字用金色微光强调，呼应黄金资产属性

## 2. Art Direction

- **方向名**: 贵金属仪表盘
- **Design Style**: Modern Dark + Luxury Accent —— 深蓝底营造金融专业与专注感，金色作为贵金属资产的视觉符号，只用于关键数据与主行动
- **DNA 参数**: 圆角 rounded-lg（卡片）/ rounded-md（按钮输入）；阴影 subtle 多层微光；间距 standard（gap-4 / p-6）；字体方向 无衬线数据感 + 中文清晰易读；装饰手法 金色渐变描边、微光高光、细金线分隔
- **应用类型**: Tool + Dashboard 混合 —— 问答与计算为任务流，回测与结论为数据流

## 3. Color System

**色彩关系**: 深夜蓝基底 + 金色主强调 + 冷灰中性层级，金色只占约 8% 用于锚点与关键数据
**配色设计理由**: 深蓝底传达金融专业与数据聚焦；金色呼应"黄金"资产本身，是品牌识别与主行动色；白色与冷灰保证长文本和图表可读性；accent 用深蓝灰承接 hover 状态，避免金色泛滥
**主色推导**: 从黄金贵金属色 #D4AF37 / #B8860B 出发，统一到 hsl(45° 附近)，通过明度变化衍生金色系；深蓝基底从 #1a2332 提取，统一到 hsl(215°) 冷调，保证数据图表的沉浸感
**使用比例**: 65% 深蓝中性（bg + card 变体）/ 27% 辅助冷灰与白（text + border + accent）/ 8% 金色 primary；金色只用于主按钮、关键数字、激活 tab 指示线、图表主曲线，禁止铺满

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(215 35% 9%) | 深夜蓝页面背景，金融仪表盘基底 |
| card | `--card` | `bg-card` | hsl(215 30% 14%) | 卡片与面板，比 bg 亮 5%，微浮起 |
| text | `--foreground` | `text-foreground` | hsl(210 20% 96%) | 标题与正文，高对比白 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(215 15% 65%) | 辅助说明、标签、元信息 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(45 75% 55%) | 金色主色，对应黄金资产，CTA 与关键数据 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(215 40% 10%) | 金色上的深文字，保证可读性 |
| accent | `--accent` | `bg-accent` | hsl(215 25% 20%) | hover/focus 浅底、选中态、菜单高亮 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(45 70% 70%) | accent 上的浅金文字与图标 |
| border | `--border` | `border-border` | hsl(215 20% 25%) | 卡片与输入框边界，冷灰低对比 |

**语义色提示**: 成功（上行/正向收益）hsl(150 50% 50%)，三态：bg hsl(150 40% 18%) / border hsl(150 45% 35%) / text hsl(150 60% 65%)；警告（风险提示）hsl(35 80% 55%)，三态：bg hsl(35 50% 18%) / border hsl(35 60% 35%) / text hsl(35 85% 65%)；错误（极端风险）hsl(0 65% 58%)，三态：bg hsl(0 40% 18%) / border hsl(0 50% 35%) / text hsl(0 75% 68%)；所有语义色饱和度与 primary 金色对齐 ±15%，避免状态色刺眼压过主品牌

## 4. 字体与节奏

- **font-display**: Space Grotesk —— 几何无衬线，数字与标题有精密仪表感，呼应金融数据与策略量化气质
- **font-body**: Noto Sans SC + Inter fallback —— 中文清晰易读，英文与数字现代克制，适合长文本与表单
- **字号**: H1 text-4xl ~ text-5xl；H2 text-xl ~ text-2xl；body text-base；muted text-sm；关键数据数字 text-3xl ~ text-5xl 金色加粗
- **圆角**: 中 —— 卡片 rounded-lg，按钮输入 rounded-md，徽章滑块 rounded-full，保持专业不冰冷

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导，顶部导航 + 四页单页切换（问答 / 配置 / 回测 / 结论）
- **Page / Section Order**: 智能问答首页 → 行业配置计算器 → 策略回测看板 → 研究结论，与需求 1:1 对齐
- **Standard Content Zone**: 后台型 max-w-6xl + `mx-auto`，回测页图表区可扩展到 max-w-7xl
- **Shell / Frame Alignment**: 顶部导航栏全宽，内容区与导航内联容器同宽对齐
- **Padding & Rhythm**: `px-4 md:px-6 lg:px-8 py-8 md:py-12`，卡片内边距 p-6，8px 倍数节奏
- **Full-bleed Zones**: 导航栏、页脚背景全宽；首页问答 Hero 区背景渐变可全宽，内部输入与结果受内容区约束
- **Local Narrowing**: 问答输入区、计算器表单区收窄到 max-w-3xl 居中；回测图表全宽铺开
- **Overflow Strategy**: 回测表格、行业数据长列表使用 `overflow-x-auto`；饼图与曲线自适应容器
- **Flexibility Boundary**: 允许移动端卡片堆叠、滑块标签换行、图表高度自适应；不允许切换主色、圆角系统或金色使用规则

## 6. 视觉与动效

- **装饰**: 金色微光描边、细金线分隔、卡片顶部 1px 高光
- **阴影/边界**: 轻 —— 卡片用 `shadow-lg` 深色软阴影 + 内顶 1px 高光模拟浮起；输入与按钮用边框 + focus 金色外发光
- **动效**: 精致克制 —— hover 时卡片微亮 + 金色描边微显；数字变化用 300ms 缓动过渡；图表入场 600ms 淡入绘制；页面切换 200ms 淡入淡出

## 7. 组件原则

- 按钮、输入、滑块、下拉、tab、卡片必须有 Default / Hover / Active / Focus-visible / Disabled 状态
- Primary 金色只用于主 CTA（计算 / 提问 / 切换结果）；次级操作用 outline + 冷灰边框；ghost 按钮用 accent 底承接 hover
- 关键指标数字用金色大号字 + 左侧细金线装饰，形成"仪表盘读数"视觉
- 滑块轨道用深蓝灰，填充进度用金色渐变，thumb 用金色圆 + 微光
- 空状态、加载态延续深蓝 + 金色骨架屏，不回退到默认浅色样式

## 8. Image Direction

- **Image Role**: 背景氛围图 + 首页 Hero 视觉层
- **Image Art Direction**: 深色金融场景，黄金金属质感与股票K线/数据网格的抽象融合；低角度侧光，金色高光从右上角洒下；深蓝暗影中隐约可见网格线与走势曲线；整体低调不抢戏，作为文字与输入框的景深背景；前景文字区有半透明深色蒙版保证可读性
- **Image Prompt Keywords**: abstract gold bars and stock chart grid, dark navy blue background, warm golden side lighting from top right, metallic gold texture, subtle data grid lines, cinematic depth of field, minimalist financial aesthetic, soft glow, high detail, 8k, dark mood
- **Image Avoidance**: 避免真实金条金币堆的俗套素材图、避免商务人物握手照片、避免高饱和霓虹赛博风、避免无意义彩色渐变、避免卡通插画风格

## 9. Anti-patterns

- **Gold rush**: 金色铺满按钮、图标、边框、链接、标题，失去贵金属的稀缺感；把金色收回到 CTA、关键数字、主曲线、激活指示
- **Light mode drift**: 回退到默认白底蓝按钮的 SaaS 通用感；保持深蓝仪表盘基底作为全站统一语言
- **Chart chaos**: 图表用 5 种以上高饱和色；限制在金 + 白 + 冷灰 + 1 个语义色，主曲线金色最亮
- **Invisible interaction**: 深色模式下 focus-visible 丢失；所有交互元素必须有金色外发光或描边的键盘可见状态
- **Status color drift**: 红绿色涨跌提示饱和度过高，压过金色主品牌；语义色饱和度与 primary 对齐 ±15%
- **Mono-density**: 所有卡片同样大小同样权重；关键指标卡用更大字号 + 金色强调，形成视觉层级
